# AWS ハンズオン手順書

## 概要

本手順書はAWSアカウントの初期セキュリティ設定から、S3・Lambda・EC2・Dockerを使ったハンズオンまでの手順をまとめたものです。

---

## 1. AWSアカウントのセキュリティ初期設定

### 1.1 rootアカウントにMFAを設定する

1. AWSコンソールにrootアカウントでログイン
2. 右上のアカウント名 → 「セキュリティ認証情報」
3. 「MFAを割り当てる」→ Google AuthenticatorやAuthyでQRコードを読み込む

### 1.2 rootアカウントの不要な認証情報を削除する

「セキュリティ認証情報」ページで以下を確認・削除する。

| 項目 | 対応 |
|---|---|
| アクセスキー | 存在する場合は削除 |
| X.509デジタル署名用証明書 | 存在する場合は削除 |
| CloudFrontキーペア | 存在する場合は削除 |

### 1.3 IAMユーザーおよびロールへの請求情報アクセスを有効化する

1. 右上のアカウント名 → 「アカウント」
2. 「IAMユーザーおよびロールへの請求情報へのアクセス」→「編集」
3. 「IAMアクセスのアクティブ化」にチェック → 「更新」

---

## 2. IAMユーザーの作成

### 2.1 IAMユーザーを作成する

1. コンソール上部の検索バーに `IAM` と入力 → 選択
2. 左メニュー「ユーザー」→「ユーザーの作成」
3. 以下を設定する

| 項目 | 設定値 |
|---|---|
| ユーザー名 | 任意（例：`yuusuke-admin`） |
| コンソールアクセス | 有効化 |
| パスワード | カスタムパスワードで設定 |
| 次回サインイン時にリセット | オフ |
| ポリシー | `AdministratorAccess`をアタッチ |

4. 作成後に表示されるサインインURLをメモする

### 2.2 IAMユーザーにMFAを設定する

1. IAMユーザーでログイン
2. 右上のアカウント名 → 「セキュリティ認証情報」
3. 「MFAデバイスの割り当て」→ 認証アプリでQRコードを読み込む

---

## 3. 請求アラートの設定

1. 検索バーに `Budgets` と入力 → 「AWS Budgets」を選択
2. 「予算を作成」→「テンプレートを使用する」→「月次コスト予算」
3. 以下を設定する

| 項目 | 設定値 |
|---|---|
| 予算名 | 任意（例：`monthly-budget`） |
| 予算額 | $10 |
| 通知先メールアドレス | 自分のメールアドレス |

4. 「予算を作成する」

---

## 4. VPCの作成

1. 検索バーに `VPC` と入力 → 「VPC」を選択
2. 左メニュー「お客様のVPC」→「VPCを作成」
3. 以下を設定する

| 項目 | 設定値 |
|---|---|
| 作成するリソース | VPCなど |
| 名前タグ | `handson` |
| IPv4 CIDRブロック | `10.0.0.0/16` |
| アベイラビリティゾーン数 | 1 |
| パブリックサブネット数 | 1 |
| プライベートサブネット数 | 0 |
| NATゲートウェイ | なし |
| VPCエンドポイント | なし |

4. 「VPCを作成」

---

## 5. S3静的ホスティング

### 5.1 S3バケットを作成する

1. 検索バーに `S3` と入力 → 「S3」を選択
2. 「バケットを作成」
3. 以下を設定する

| 項目 | 設定値 |
|---|---|
| バケット名 | グローバルでユニークな名前（例：`handson-yuusuke-20260622`） |
| リージョン | アジアパシフィック（東京）ap-northeast-1 |
| パブリックアクセスのブロック | すべてのチェックを外す・確認チェックを入れる |

4. 「バケットを作成」

### 5.2 静的ウェブサイトホスティングを有効にする

1. 作成したバケット → 「プロパティ」タブ
2. 「静的ウェブサイトホスティング」→「編集」
3. 以下を設定する

| 項目 | 設定値 |
|---|---|
| 設定 | 有効にする |
| ホスティングタイプ | 静的ウェブサイトをホストする |
| インデックスドキュメント | `index.html` |

4. 「変更の保存」

### 5.3 バケットポリシーを設定する

1. 「アクセス許可」タブ → 「バケットポリシー」→「編集」
2. 以下のJSONを貼り付ける（`YOUR-BUCKET-NAME`を実際のバケット名に変更）

```json
{
  "Version": "2012-10-17",
  "Statement": [
    {
      "Sid": "PublicReadGetObject",
      "Effect": "Allow",
      "Principal": "*",
      "Action": "s3:GetObject",
      "Resource": "arn:aws:s3:::YOUR-BUCKET-NAME/*"
    }
  ]
}
```

3. 「変更の保存」

### 5.4 index.htmlをアップロードして確認する

以下の内容で`index.html`を作成する。

```html
<!DOCTYPE html>
<html lang="ja">
<head>
  <meta charset="UTF-8">
  <title>AWS S3 ハンズオン</title>
</head>
<body>
  <h1>Hello AWS S3!</h1>
  <p>S3の静的ホスティングが動いています。</p>
</body>
</html>
```

1. バケットの「オブジェクト」タブ → 「アップロード」→ `index.html`を選択 → 「アップロード」
2. 「プロパティ」タブ → 「静的ウェブサイトホスティング」→ バケットウェブサイトエンドポイントのURLをブラウザで開く
3. 「Hello AWS S3!」が表示されれば成功

## 6. Lambda関数の作成

### 6.1 Lambda関数を作成する

1. 検索バーに `Lambda` と入力 → 「Lambda」を選択
2. 「関数の作成」→「一から作成」
3. 以下を設定する

| 項目 | 設定値 |
|---|---|
| 関数名 | `handson-function` |
| ランタイム | Python 3.12 |
| アーキテクチャ | x86_64 |

4. 「関数の作成」

### 6.2 テスト実行する

デフォルトのコードをそのまま使う。

```python
import json

def lambda_handler(event, context):
    return {
        'statusCode': 200,
        'body': json.dumps('Hello from Lambda!')
    }
```

1. 「テスト」ボタン → 「新しいテストイベントを作成」
2. イベント名：`test`→「保存」→「テスト」
3. レスポンスに`"Hello from Lambda!"`が表示されれば成功

---

## 7. EC2インスタンスの作成とSSH接続

### 7.1 EC2インスタンスを作成する

1. 検索バーに `EC2` と入力 → 「EC2」を選択
2. 「インスタンスを起動」
3. 以下を設定する

| 項目 | 設定値 |
|---|---|
| 名前 | `handson-ec2` |
| AMI | Amazon Linux 2023 |
| インスタンスタイプ | t2.micro（無料枠対象） |
| キーペア | 新しいキーペアを作成（名前：`handson-key`、形式：.pem） |
| VPC | `handson` |
| サブネット | パブリックサブネット |
| パブリックIPの自動割り当て | 有効化 |
| セキュリティグループ | 新規作成・SSHのソースは「自分のIP」 |

4. 「インスタンスを起動」
5. ダウンロードされた`handson-key.pem`は大切に保管する

### 7.2 SSH接続する

```bash
# .pemファイルの権限を設定
chmod 400 ~/Downloads/handson-key.pem

# SSH接続（<パブリックIP>はEC2コンソールで確認）
ssh -i ~/Downloads/handson-key.pem ec2-user@<パブリックIP>
```

## 8. NginxをEC2上で起動する

```bash
# パッケージを更新
sudo dnf update -y

# Nginxをインストール
sudo dnf install -y nginx

# Nginxを起動
sudo systemctl start nginx

# 自動起動を有効化
sudo systemctl enable nginx
```

セキュリティグループにポート80を追加する。

1. EC2コンソール → 対象インスタンス → 「セキュリティ」タブ → セキュリティグループをクリック
2. 「インバウンドのルールを編集」→「ルールを追加」
3. タイプ：`HTTP`、ソース：`0.0.0.0/0` → 「ルールを保存」

ブラウザで`http://<パブリックIP>`を開き、Nginxのページが表示されれば成功。

---

## 9. DockerをEC2にインストールする

```bash
# Dockerをインストール
sudo dnf install -y docker

# Dockerを起動
sudo systemctl start docker

# 自動起動を有効化
sudo systemctl enable docker

# ec2-userをdockerグループに追加
sudo usermod -aG docker ec2-user

# 一度ログアウトして再接続（グループ変更を反映）
exit
```

再度SSH接続後、動作確認する。

```bash
docker --version
```

### 9.1 NginxコンテナをDockerで起動する

```bash
# NginxコンテナをポートにマッピングSして起動
docker run -d -p 8080:80 --name my-nginx nginx
```

セキュリティグループにポート8080を追加する。

1. 「インバウンドのルールを編集」→「ルールを追加」
2. タイプ：`カスタムTCP`、ポート：`8080`、ソース：`0.0.0.0/0` → 「ルールを保存」

ブラウザで`http://<パブリックIP>:8080`を開き、Nginxのページが表示されれば成功。

### 9.2 Dockerの基本操作

```bash
# 起動中のコンテナ一覧
docker ps

# 全コンテナ一覧（停止中含む）
docker ps -a

# コンテナを停止
docker stop my-nginx

# コンテナを再起動
docker start my-nginx

# コンテナの中に入る
docker exec -it my-nginx bash

# コンテナを停止・削除
docker stop my-nginx && docker rm my-nginx

# 使っていないイメージを削除
docker image prune -a
```

## 10. EC2インスタンスの停止

学習終了後はEC2インスタンスを停止して無料枠の消費を防ぐ。

1. EC2コンソール → インスタンス一覧 → 対象インスタンスを選択
2. 「インスタンスの状態」→「インスタンスを停止」

> 削除（終了）ではなく停止にすること。終了するとインスタンスが削除される。
