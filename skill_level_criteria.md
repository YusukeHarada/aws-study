# AWS スキルレベル基準表

## レベル定義

| レベル | 名称 | 一般的な定義 |
|--------|------|-------------|
| Lv1 | **入門** | コンソールの指示に従って操作できる。エラーが出たら詰まる。 |
| Lv2 | **基礎** | 基本ユースケースを自力で実装・設定できる。公式ドキュメントを参照しながら作業できる。 |
| Lv3 | **応用** | 複数サービスを組み合わせてアーキテクチャを設計・実装できる。トラブルシューティングができる。 |
| Lv4 | **実践** | 本番運用・セキュリティ・コスト最適化まで考慮した設計・運用ができる。他者にレビューできる。 |

---

## サービス別 スキルレベル基準

### IAM

| Lv | 基準 |
|----|------|
| Lv1 | コンソールから IAM ユーザーを作成し MFA を設定できる |
| Lv2 | グループ・ポリシーを作成し、最小権限でユーザーに割り当てられる |
| Lv3 | IAM ロールを設計し、Lambda/EC2 から AssumeRole で使用できる |
| Lv4 | SCPs (Service Control Policies) や Permission Boundary を使った多層防御ができる |

### VPC / ネットワーク

| Lv | 基準 |
|----|------|
| Lv1 | デフォルト VPC の存在を理解し、コンソールからカスタム VPC を作成できる |
| Lv2 | パブリック / プライベートサブネット・ルートテーブル・IGW を正しく構成できる |
| Lv3 | NAT GW、VPC Peering、セキュリティグループとネットワーク ACL の違いを理解して設計できる |
| Lv4 | Transit Gateway や PrivateLink を使ったマルチ VPC アーキテクチャを設計できる |

### S3

| Lv | 基準 |
|----|------|
| Lv1 | バケットを作成しファイルをアップロード・公開できる |
| Lv2 | バケットポリシー・静的ホスティング・バージョニングを設定できる |
| Lv3 | CloudFront と組み合わせた HTTPS 配信・署名付き URL・ライフサイクルポリシーを設定できる |
| Lv4 | クロスリージョンレプリケーション・Object Lock・S3 Analytics でコスト最適化できる |

### Lambda

| Lv | 基準 |
|----|------|
| Lv1 | コンソールから関数を作成しテスト実行できる |
| Lv2 | API Gateway トリガーを設定し、環境変数・タイムアウト・メモリを調整できる |
| Lv3 | DynamoDB / SQS / S3 と連携した処理を設計・実装できる。エラーハンドリング・DLQ を設定できる |
| Lv4 | コールドスタート対策（Provisioned Concurrency）・X-Ray 計装・Lambda Power Tuning でチューニングできる |

### API Gateway

| Lv | 基準 |
|----|------|
| Lv1 | REST API を作成し Lambda に接続してテストできる |
| Lv2 | ステージ・デプロイ・CORS 設定・リクエストバリデーションを設定できる |
| Lv3 | カスタムドメイン・使用量プラン・API キー・オーソライザー (Cognito/Lambda) を設定できる |
| Lv4 | WebSocket API・HTTP API vs REST API の選択基準を理解し、本番グレードの API 設計ができる |

### DynamoDB

| Lv | 基準 |
|----|------|
| Lv1 | テーブルを作成し、コンソールから項目の追加・参照ができる |
| Lv2 | Lambda から SDK を使って Put/Get/Query/Delete 操作ができる |
| Lv3 | GSI (グローバルセカンダリインデックス) を設計し、効率的なクエリパターンを実装できる |
| Lv4 | キャパシティプランニング・DAX (キャッシュ)・DynamoDB Streams を活用した設計ができる |

### EC2

| Lv | 基準 |
|----|------|
| Lv1 | インスタンスを起動し SSH 接続できる |
| Lv2 | セキュリティグループ・キーペア・ユーザーデータを使ってアプリを起動できる |
| Lv3 | AMI の作成・起動テンプレート・Auto Scaling Group を設定できる |
| Lv4 | スポットインスタンス・Reserved/Savings Plans でコスト最適化した設計ができる |

### Docker / コンテナ

| Lv | 基準 |
|----|------|
| Lv1 | Docker をインストールし、既存イメージを pull・run できる |
| Lv2 | Dockerfile を書いてイメージをビルドし ECR にプッシュできる |
| Lv3 | ECS タスク定義を作成し Fargate でコンテナを ALB 経由で公開できる |
| Lv4 | マルチコンテナ構成・Blue/Green デプロイ・コンテナのセキュリティ設定ができる |

### CloudFormation / IaC

| Lv | 基準 |
|----|------|
| Lv1 | 既存のテンプレートを使ってスタックを作成・削除できる |
| Lv2 | 基本リソース（VPC, EC2, S3）のテンプレートを自分で書ける |
| Lv3 | ネストされたスタック・パラメータ・条件分岐を使った再利用可能なテンプレートを書ける |
| Lv4 | StackSets でマルチアカウント・マルチリージョンのデプロイを自動化できる |

### CloudWatch / 監視

| Lv | 基準 |
|----|------|
| Lv1 | CloudWatch コンソールでログを確認できる |
| Lv2 | カスタムメトリクス・アラームを設定し SNS で通知できる |
| Lv3 | ダッシュボード・ログインサイト・メトリクスフィルタで運用監視基盤を構築できる |
| Lv4 | X-Ray・ServiceLens・Container Insights を組み合わせた総合的な可観測性を設計できる |
